SCORE API

Overview

The SCORE API provides programmatic access to satellite observation data, allowing you to submit and retrieve observations through a RESTful interface. The API is rate-limited to 10 requests per second.

An API key is required to upload observations via the API - to request an API key, please click here.

Full API Documentation

The full OpenAPI documentation is available at https://score.cps.iau.org/api/docs.

Base URL

https://score.cps.iau.org/api

Available Endpoints

Endpoint Description Parameters
/observations/{id} Get details for a specific observation observation_id (int)
/observations List all observations (paginated) offset, limit
/observations/recent Get most recent observations offset, limit (optional)
/observations/stats Get observation statistics None
/observations/search Search observations with filters satellite_number, start_date, end_date, min_magnitude, max_magnitude
/satellites/{number}/observations Get observations for a specific satellite satellite_number (NORAD ID)
/satellites/{number} Get details for a specific satellite satellite_number (NORAD ID)
/satellites List all satellites name (optional - to filter)
/upload Upload observations observations (list of dictionaries)
/upload/{batch_id}/status Get the status of a batch upload batch_id (string)

Example Usage

Data Retrieval Examples

1. Search Observations by Date Range:

GET /api/search?start_date=2025-01-01T00:00:00.000Z&end_date=2025-01-31T23:59:59.999Z

Returns all observations between January 1st and January 31st, 2025.

2. Search by Satellite Number:

GET /api/search?satellite_number=58013

Returns all observations for satellite with NORAD ID 58013.

3. Get Recent Observations:

GET /api/observations/recent?limit=10

Returns the 10 most recent observations in the database.

Response Format

All responses include:

  • items: Array of observation results
  • count: Total number of matching observations
Uploading Observations

Uploading observations requires an API key, which can be requested here. This key should be used for both the initial upload and for checking the status of the upload.

Include the API key in the Authorization header of the request:

Example: cURL
curl -X POST https://score.cps.iau.org/api/upload \
                                -H "Authorization: Bearer {api-key}" \
                                -H "Content-Type: application/json" \
                                -d @observations.json
Example: Python
import requests

api_key = "{api-key}"
headers = {
    "Authorization": f"Bearer {api_key}",
    "Content-Type": "application/json"
}

response = requests.post(
    "https://score.cps.iau.org/api/upload",
    headers=headers,
    json=observation_data
)
Security Best Practices:
  • Store the key securely (use environment variables or secrets management)
  • Never commit the key to version control
  • Always use HTTPS for API requests
  • Report compromised keys immediately

Observation Response Format

All responses are returned in JSON format. The API supports pagination for endpoints that return multiple results. Each observation includes:

  • Basic observation details (time, magnitude, uncertainties)
  • Position measurements (RA, Dec, range, range rate)
  • Observer information (instrument, mode, filter)
  • Location data (latitude, longitude, altitude)
  • Associated satellite information
